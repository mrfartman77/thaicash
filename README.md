# ThaiCash

On-device iOS app that shows the **real, all-in cost of converting USD → THB** across every method, normalized to net baht received, so you can pick the cheapest option for your situation. Thailand-specific. **Fully free — no accounts, no subscriptions, no in-app purchases.**

## What it does
Enter the baht you need ("I need ฿40,000" — or enter dollars and it converts at the live rate) and see every method ranked by all-in cost, inside output groups that aren't interchangeable:
- **Cash in hand** — exchange booth, or an ATM withdrawal (Schwab debit vs Wise vs Revolut vs your own debit/credit — one Home row, ranked by card inside; no brand is featured, the data decides)
- **THB in your bank** — Wise / Remitly / Revolut / Xe / OFX, ranked by what lands
- **Crypto → Thai bank** — sell USDT on the licensed Thai venues (Binance TH / Bitkub / Bitazza) and withdraw baht; published fees are identical, the live book decides

Each method opens into a detail screen — the booth's carries a live directory of Bangkok's best-known chains with today's scraped board rates and Apple Maps links. A 7-day rate chart sits on top. Your setup (your bank's fees, card terms, days in Thailand) persists locally.

## Architecture
- **Pure engine** (`Engine/`) — `Engine.compare()` takes catalog + profile + target amount + live rate → grouped, ranked `MethodResult`s. No I/O, fully unit-testable.
- **Catalog** (`Resources/catalog.json` + `Services/CatalogService`) — every fee / margin / cap / limit lives in data, fetched from a CDN with a **bundled seed + cache** fallback and a schema-version guard. Fee updates ship *without* an App Store release; only structural changes bump the schema.
- **Rate** (`Services/RateService`) — live USD→THB from a **keyless** API (open.er-api primary, Frankfurter fallback — both verified), cached, offline-safe, with a manual-entry fallback, plus the 7-day history series for the chart. Daily granularity is plenty: intraday FX moves are ~10× smaller than the fee differences we surface.
- **Profile** (`Models/`) — your fees / cards / booth margin / funding source / days-in-Thailand. Saved locally on every change (`AppModel.update`). No backend, no accounts.

## Design
Dark-only "private bank" language: champagne gold (`Color.bahtGold`) as the single brand accent, sage for positives, straight SF Pro with tabular digits, hairline-bordered cards on near-black. App icon = gradient-gold ฿ ("Gilded") in `Resources/Assets.xcassets`.

## Folder layout
`App/` · `Models/` · `Engine/` · `Services/` · `Views/` · `Resources/`

## Build & run
1. Open `ThaiCash.xcodeproj` → pick an iPhone simulator → ⌘R. (iOS 17+.)
2. The project is generated by **XcodeGen** from `project.yml` — after adding/removing files, run `xcodegen generate --spec project.yml` (or just manage the project directly in Xcode and retire the yml).
3. Before shipping: point `CatalogService.remoteURL` at your real hosted catalog (until then the bundled seed + cache serve everything).

## Conventions
- All money math uses `Decimal`, never `Double`.
- Engine works in **THB-target mode** ("I need ฿X") — costs are computed *to obtain* the target, so rankings are near rate-independent (only fixed-USD fees and the booth's absolute quote move with the rate).
